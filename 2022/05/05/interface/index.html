Lambda表达式

Lambda表达式是什么

Lambda表达式其实就是一种匿名函数，其可看作匿名内部类的简化

为什么用Lambda表达式

Lambda表达式允许将一个函数作为另外一个函数的参数； 我们可以把 Lambda 表达式理解为是一段可以传递的代码（将代码作为实参）,也可以理解为函数式编程，将一个函数作为参数进行传递。 所以Lambda能够简化我们的代码，当然前提是你的JDK版本要高于JDK8。

使用Lambda表达式的条件

只需要有函数式接口你就能使用Lambda表达式啦。

Lambda表达式的语法

无返回值时:(parameters)-> { 语句1;语句2;}
有返回值时:(parameters) -> { 语句1;语句2; return result;}
表达式左边是参数列表，参数名是可以随意起的（最好有实际意义），右边则是Lambda要实现功能的代码， 是主体部分，Lambda表达式内的参数和定义的变量不能和局部变量名相同。 其中的参数列表是依据接口中方法的参数并且与之一一对应，有无返回值也是看接口中方法是否有返回值，类型保持一致。
而上述的语法体仍能化简:
1.由于函数式接口的方法唯一(不考虑默认方法)，所以参数列表中变量类型可以省略，由编译器进行推导。
(param1,param2)-> { 语句1;语句2;}
(param1,param2) -> { 语句1;语句2; return result;}
2.若参数列表的参数只有1个参数，则可以省略()，但是多参数和没有参数时是不能省略()的
param-> { 语句1;语句2;}
param -> { 语句1;语句2; return result;}
3.若语句只有一条且无返回值则可以省略{}
(parameters)-> 语句1;
4.若主体只有一个返回语句则可省略return关键字
(param1,param2) -> result;

Lambda具体该如何使用

1.在主体中实现功能

即将想要实现的功能代码直接写在主体中，看起来就像一个普通的方法，我们来想一下一个普通的方法是怎么写的， 再看一下lambda的语法，只不过是一个没有名字的匿名函数，它的使用方法就是调用接口的方法。

2.方法引用

要传递给Lambda表达式体的操作，已经有实现的方法了，就可以使用方法引用(方法签名需一致)。方法引用可以理解为一种特殊的方法调用形式。
方法引用的语法:对象::实例方法名、类::静态方法名、类::实例方法名
想要熟练地使用这些方法引用，首先你要熟悉前面的最基本的Lambda表达式的用法，并掌握函数式编程的方法和常用接口。

3.构造器引用

与函数式接口相结合，自动与函数式接口中的方法匹配，可以把构造器中的引用赋值给定义的方法。需要注意参数列表与构造器的参数列表要相一致。
构造器引用语法:类名::new

Lambda表达式的作用域

Lambda表达式的局部变量可以访问局部变量，局部引用，静态变量和实例变量

Lambda表达式访问局部变量时有限制，必须时最终变量或实际上的最终变量，即无法被重新赋值的变量

Lambda表达式访问局部引用、静态变量和实例变量时没有限制

对于上述情况的解释:为什么只对局部变量有限制？因为局部变量是存储在栈帧中的，其线程是私有的，若允许Lambda访问，则可能导致私有数据并发访问，从而导致线程不安全。而对于引用变量、实例变量 和静态变量，他们都是储存在堆中的，堆是线程共享的，因此允许访问。对于局部变量而言，其线程无法看到其他线程对自己数据的操作，从而无法保证数据的同步，所以需要使用final进行修饰。而共享的线程 是能够看到其他线程对数据的操作修改的，所以可以直接访问。

函数式接口

函数式接口是什么

函数式接口就是接口的一种，只不过它有且仅有一个抽象方法。

如何使用函数式接口

  自定义函数式接口

我们该如何写一个函数式接口呢，其实非常简单，因为我们已经知道接口怎么写了，那么我们就知道函数式接口该怎么写，只要接口有唯一的抽象方法就是函数式接口了。只不过在平时写函数式接口时 我们习惯加上@FunctionalInterface注解，这个注解就是检查我们的接口是否符合函数式接口的要求的，以防我们犯错。

  函数式接口作为参数

函数式接口作为参数其实并没有什么特别，就和普通的参数一样，只不过函数式接口可能更加庞大，所以要配合Lambda表达式使用。 如果你还是不理解函数式接口作为参数，那么你可以想一下类作为参数，其实这个也类似，只不过你在把它当作参数时并没有具体的实现 所以也就意味着你可以根据需求得到多种不同的实现

  函数式接口作为返回值

看完了函数式接口作为参数，那么这个就很好理解了吧。同样的想一下类作为返回值的情况就不难理解了

四个常用的函数式接口

在java.lang.function包下，有很多支持函数式编程的接口，可以说是非常方便，常用的接口有四个，我们下面将主要介绍它们的用法。 此包下更多接口使用请见 官方文档。

  Supplier接口

Supplier接口 有返回值 无参数,返回值类型为泛型需要你根据需要自己选择。
对于该接口的方法的简单介绍:
该接口是生产型接口，泛型指定数据后，根据我们所写的逻辑返回值给我们使用。
无参方法 T get():获得结果

  Consumer接口

Consumer接口 有一个参数 无返回值,参数类型为泛型,根据自己选择。
对于该接口的方法的简单介绍:
该接口为消费型接口,其消费的数据由泛型决定
void accept(T t):对给定的参数执行操作
default Consumer <T> andThen(Consumer after):返回一个组合的操作，依次执行操作

  Predicate接口

Predicate接口 有一个参数 有返回值,参数为泛型,返回值为boolean类型。
对于该接口的方法的简单介绍:
该接口常用于判断参数是否满足给定条件
boolean test(T t):对给定的参数进行判断，其逻辑由使用者实现(一般使用Lambda表达式实现)
default Predicate <T> negate():返回给定判断逻辑的否定
default Predicate <T> and(Predicate other):返回组合判断，对于短路与
default Predicate <T> or(Predicate other):返回组合判断，对应短路或

  Function接口

* Function接口 有一个参数 有返回值,参数和返回值均为泛型。
* 对于该接口方法发简单介绍:
* 此接口通常用于对参数进行处理、转换，然后返回值
* R apply(T t):将此函数应用于给定的参数
* default <V> Function andThen(Function after):返回一个组合函数，依次将值应用于函数中

如果能掌握这四个常用的函数式接口的话，那么Java.lang.function包下其他的函数式接口学起来就非常简单了
